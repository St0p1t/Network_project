# Resonance

Браузерная генеративная инсталляция: экосистема из частиц, существ и растений реагирует на звук в реальном времени. Говори, пой, хлопай — мир меняется. Загрузи музыку и мир будет танцевать в такт.

---

## Архитектура

```
Браузер                               Python сервер (FastAPI)
──────────────────────────────────    ──────────────────────────────────────
Web Audio API → анализ звука          asyncio sim-loop @ 30 fps
  rms, bass, mid, high,               ├── world/boids.py   (numpy, O(n²))
  pitch, onset, silence               ├── world/flora.py   (ветвящиеся нити)
        │                             └── world/events.py  (частицы + кольца)
        │  JSON {"type":"audio",...}        │
        │ ─────────────────────────────→   │  обновляет current_audio
        │                                  │
        │  binary frame (4.9 KB/кадр)  ←──┤  позиции boids (int16 × 4 на boid)
        │  JSON {"type":"fe",...}       ←──┘  флора + события
        ↓
  env_gl.js    — WebGL фон (GLSL шейдер)
  main.js      — Canvas 2D: boids, флора, частицы
```

### Почему один воркер

Симуляция живёт в оперативной памяти. Несколько uvicorn-воркеров создали бы несколько независимых миров. Один воркер + asyncio — правильное решение для stateful real-time приложения.

### Протокол бинарного фрейма (boids)

```
[uint32 count] + count × [int16 px][int16 py][uint16 angle][uint8 energy]
```

Итого на 700 boids: **4 + 700 × 7 = 4 904 байта** за кадр.  
При 30 fps: ~148 KB/с на клиента — в 20× меньше, чем JSON.

## Способ 1 — Локальный запуск

### Шаг 1. Создай виртуальное окружение и установи зависимости

```bash
cd Network_project

python -m venv venv
source venv/bin/activate        # Linux / macOS
# venv\Scripts\activate         # Windows

pip install -r server/requirements.txt
```

### Шаг 2. Запусти сервер

```bash
uvicorn server.main:app --host 0.0.0.0 --port 8000 --reload
```

### Шаг 3. Открой браузер

```
http://localhost:8000
```

`localhost` — безопасный источник, микрофон работает без HTTPS.

`--reload` включает автоперезагрузку при изменении Python-файлов.

---

## Способ 2 — Деплой на сервере (Docker + Caddy)

Caddy автоматически получает сертификат Let's Encrypt. HTTPS обязателен для доступа к микрофону с внешних устройств.

### Шаг 1. Направь DNS на сервер

```
resonance.example.com  A  <IP сервера>
```

### Шаг 2. Убедись, что порты 80 и 443 открыты

### Шаг 3. Отредактируй `Caddyfile`

```caddy
resonance.example.com {
    reverse_proxy app:8000
    encode gzip
}

:80 {
    redir https://{host}{uri} permanent
}
```

### Шаг 4. Собери и запусти

```bash
docker compose up --build -d
```

Сертификат выпускается автоматически при первом запросе и обновляется без вмешательства. Хранится в томе `caddy_data`.

### Управление контейнерами

```bash
docker compose logs -f          # следить за логами
docker compose down             # остановить
docker compose up -d            # запустить снова (без пересборки)
docker compose up --build -d    # пересобрать и запустить
```

## Управление

| Клавиша / элемент | Действие |
|---|---|
| `D` | Показать/скрыть дебаг-оверлей (RMS, bass, mid, high, pitch, onset, silence, fps, WS) |
| `F` | Полноэкранный режим |
| `R` | Сбросить мир (новые позиции, новое число кластеров) |
| Кнопка **↺ Сбросить** | То же, что `R` |
| Кнопка **← Меню** | Вернуться на стартовый экран (мир продолжает работать) |
| Ползунок **Чувствительность** | 1–10; масштабирует все аудио-значения перед отправкой на сервер. Помогает подстроиться под тихий или громкий микрофон |
| **⏸ / ▶** | Пауза / воспроизведение музыки (только в музыкальном режиме) |

## Как звуки влияют на мир

| Звук | Эффект |
|---|---|
| **Резкий звук** (хлопок, удар, слог, атака ноты) | Стая разлетается в случайные стороны; вспышка частиц + ударная волна; вырастают ветви |
| **Громкость** | Скорость и хаос движений; пульсация фона; длина ветвей |
| **Бас (0–250 Гц)** | Расширяет личное пространство существ, разбивает стаю на группы |
| **Голос и речь (250–2000 Гц)** | Переливы и мерцание фона |
| **Высокие звуки (2–8 кГц)** | Искры и блёстки на фоне |
| **Тональность** | Цвет всего в мире: низкий тон → фиолетовый, высокий → голубой |
| **Тишина** | Существа сходятся к 3–10 блуждающим точкам-кластерам (число случайное при каждом старте/сбросе); в глубокой тишине медленно растут ветви |

## Источники звука

**Голос (🎤)** — браузер запрашивает доступ к микрофону. При отказе мир переходит в автономный режим: синусоидальные аудио-значения симулируют живой звук.

**Музыка (🎵)** — два варианта:
- Локальный файл: drag & drop или выбор через диалог (MP3, WAV, OGG, FLAC)
- Поиск на [Internet Archive](https://archive.org): браузер отправляет запрос на свой сервер, сервер проксирует запрос к archive.org, возвращает список треков с прямыми ссылками на MP3

---

## Конфиденциальность

Сырой аудиосигнал никогда не покидает браузер — на сервер уходят только семь чисел: `rms, bass, mid, high, pitch, onset, silence`. WebSocket-трафик шифруется через WSS (HTTPS). Сервер не пишет данные на диск.

## Настройка

**Количество boids** — в `server/main.py`:

```python
boids = Boids(count=400)   # облегчённый вариант
```

**Частота симуляции** — в `server/main.py`:

```python
target = 1.0 / 20.0   # 20 fps вместо 30 fps
```
